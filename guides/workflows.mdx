Workflows in Regentra are automated sequences of actions triggered by events, schedules, or API calls. This guide walks you through creating, configuring, and monitoring workflows so you can automate repetitive processes and respond to changes across your stack without manual intervention.

## Creating a workflow

<Steps>
  <Step title="Navigate to Workflows">
    In your Regentra dashboard, click **Workflows** in the left sidebar.
  </Step>
  <Step title="Click New Workflow">
    Click the **New Workflow** button in the top-right corner.
  </Step>
  <Step title="Name your workflow">
    Give your workflow a clear name and an optional description so your team understands what it does at a glance.
  </Step>
  <Step title="Choose a trigger">
    Select how the workflow starts: on an event, a schedule, an incoming webhook, or manually. See [Workflow triggers](#workflow-triggers) below for details on each option.
  </Step>
  <Step title="Add actions">
    Click **Add action** to build the workflow steps. Actions can call an external API, send a notification, update a resource, or run a conditional branch.
  </Step>
  <Step title="Save and activate">
    Click **Save**, then toggle the workflow to **Active**. Regentra begins listening for the trigger immediately.
  </Step>
</Steps>

## Workflow triggers

<AccordionGroup>
  <Accordion title="Event trigger">
    Fires when a resource in Regentra is created, updated, or deleted. Select the resource type and the specific event (for example, `resource.created`) when configuring the trigger. You can filter by resource attributes to narrow which events activate the workflow.
  </Accordion>
  <Accordion title="Schedule trigger">
    Runs the workflow on a recurring schedule using cron syntax. For example, `0 9 * * 1-5` runs every weekday at 9 AM UTC. Use this trigger for periodic reports, cleanup tasks, or time-based checks.
  </Accordion>
  <Accordion title="Webhook trigger">
    Fires when Regentra receives an HTTP POST request at the workflow's unique webhook URL. Regentra generates this URL when you select the webhook trigger type. Send any JSON payload — you can reference its fields inside your workflow actions.
  </Accordion>
  <Accordion title="Manual trigger">
    Runs the workflow on demand. You can start it from the dashboard by clicking **Run now**, or call it programmatically through the REST API. Useful for one-off tasks or testing a workflow before activating an automated trigger.
  </Accordion>
</AccordionGroup>

## Workflow actions

Each step in a workflow is an action. You can chain multiple actions together, and each action can reference output from previous steps.

| Action type | Description |
|-------------|-------------|
| **Call API** | Make an HTTP request to an external endpoint. Supports GET, POST, PUT, and DELETE with custom headers and body. |
| **Send notification** | Deliver a message via Slack, email, webhook, or in-app alert. |
| **Update resource** | Modify the attributes of a Regentra resource, such as changing its status or adding metadata. |
| **Conditional branch** | Evaluate a condition and follow different paths based on the result. |
| **Delay** | Pause the workflow for a fixed duration before continuing to the next action. |

## Monitoring workflows

Regentra records every workflow run so you can track success, diagnose failures, and review what each step did.

- **Run history**: Go to **Workflows → [workflow name] → Runs** to see a list of all past executions with their status (`completed`, `failed`, `running`).
- **Step logs**: Click any run to expand the step-by-step log, including inputs, outputs, and timing for each action.
- **Error details**: Failed runs show the error message and the step where execution stopped, so you can quickly identify the cause.
- **Alerts**: Configure a notification to fire whenever a workflow fails. See the [notifications guide](/guides/notifications) for setup instructions.

## Managing workflows via API

You can create and manage workflows programmatically using the Regentra REST API. The example below creates a new workflow with a manual trigger.

```bash
curl -X POST https://api.regentra.io/v1/workflows \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Daily resource report",
    "trigger": { "type": "schedule", "cron": "0 8 * * *" },
    "actions": [
      {
        "type": "send_notification",
        "channel": "email",
        "to": "team@example.com",
        "subject": "Daily resource summary"
      }
    ]
  }'
```

<Note>
  For details on authenticating API requests, see the [API authentication guide](/api-reference/authentication).
</Note>
