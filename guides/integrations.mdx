Regentra integrations let you connect external services so you can trigger workflows and sync data across your stack. Once an integration is active, Regentra listens for events from the connected service and can act on them automatically — sending notifications, updating resources, or kicking off multi-step workflows.

## Available integrations

<CardGroup cols={2}>
  <Card title="Slack" icon="slack">
    Send messages to Slack channels when workflows complete, fail, or when resources change state. Connect by authorizing the Regentra Slack app in your workspace.
  </Card>
  <Card title="GitHub" icon="github">
    Trigger workflows when pull requests open, merge, or when issues are created. Connect using a GitHub OAuth app or personal access token.
  </Card>
  <Card title="Jira" icon="jira">
    Create or update Jira issues when Regentra workflows run. Sync resource status changes back to your project board automatically.
  </Card>
  <Card title="Email" icon="envelope">
    Send email alerts to one or more addresses when key events occur. No external service authorization required — configure your recipient addresses directly in Regentra.
  </Card>
</CardGroup>

## Setting up an integration

<Steps>
  <Step title="Go to Settings → Integrations">
    In your Regentra dashboard, open **Settings** and select **Integrations** from the left sidebar.
  </Step>
  <Step title="Select the integration">
    Browse the list of available integrations and click the one you want to add.
  </Step>
  <Step title="Authorize access">
    For OAuth-based integrations (Slack, GitHub, Jira), click **Connect** and follow the authorization flow in the pop-up window. For token-based integrations, paste your API token or credentials into the provided fields.
  </Step>
  <Step title="Configure trigger events and data mapping">
    Choose which events from the external service should fire in Regentra, and map fields between the two systems as needed.
  </Step>
  <Step title="Test the connection">
    Click **Test connection** to send a test event through the integration. Regentra confirms whether the handshake succeeded or surfaces any errors to fix.
  </Step>
</Steps>

## Managing integrations

After you set up an integration, you can view and manage all active connections from **Settings → Integrations**.

- **View status**: Each integration shows whether it is active, paused, or has encountered an error.
- **Edit configuration**: Click an integration to update trigger events, data mappings, or credentials.
- **Pause or remove**: Use the overflow menu (`⋯`) next to an integration to pause it temporarily or remove it entirely. Removing an integration stops all associated events immediately.

<Tip>
  Need to receive events from a service that isn't listed here? Use a webhook to send HTTP events directly into Regentra. See the [webhooks API reference](/api-reference/webhooks) to set one up.
</Tip>
