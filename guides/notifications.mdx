title: "Set up workflow notifications and alerts in Regentra"

---

Regentra can alert you and your team when important events occur — a workflow fails, a resource changes state, or a threshold is crossed. You choose which events matter and where the alerts go, so the right people always have the information they need without noise.

## Notification channels

<CardGroup cols={2}>
  <Card title="Slack" icon="slack">
    Post messages to any Slack channel when a subscribed event fires. Requires the Slack integration to be active in your workspace.
  </Card>
  <Card title="Email" icon="envelope">
    Send alert emails to one or more addresses. No additional integration is needed — configure recipient addresses directly in notification settings.
  </Card>
  <Card title="Webhook" icon="arrow-up-from-bracket">
    Deliver a JSON payload to any HTTP endpoint. Use this to fan out alerts to custom services, logging systems, or third-party tools.
  </Card>
  <Card title="In-app" icon="bell">
    Display alerts in the Regentra dashboard notification center. All workspace members with appropriate permissions see in-app alerts automatically.
  </Card>
</CardGroup>

## Configuring notifications

<Steps>
  <Step title="Open Notification Settings">
    In your Regentra dashboard, go to **Settings → Notifications**.
  </Step>
  <Step title="Select a channel">
    Click **Add notification** and choose your preferred channel: Slack, email, webhook, or in-app.
  </Step>
  <Step title="Configure channel settings">
    Enter the channel-specific details:
    - **Slack**: Slack channel name (for example, `#ops-alerts`)
    - **Email**: recipient email address or comma-separated list
    - **Webhook**: the destination URL that Regentra will POST to
    - **In-app**: no additional configuration required
  </Step>
  <Step title="Choose trigger events">
    Select which events activate this notification. You can subscribe to one event or many — see [Event types](#event-types) below for the full list.
  </Step>
  <Step title="Save and test">
    Click **Save**, then click **Send test** to deliver a sample payload to the configured channel. Confirm the alert arrives before relying on it in production.
  </Step>
</Steps>

## Event types

Subscribe to any combination of the following events:

| Event | Description |
|-------|-------------|
| `workflow.completed` | A workflow finished all steps without error. |
| `workflow.failed` | A workflow stopped due to an action error or timeout. |
| `resource.created` | A new resource was added to your workspace. |
| `resource.updated` | An existing resource had one or more attributes changed. |
| `resource.deleted` | A resource was permanently removed from your workspace. |

## Webhook notifications

When you configure a webhook channel, Regentra sends an HTTP POST request to your endpoint with a JSON body each time a subscribed event fires.

```json
{
  "event": "workflow.failed",
  "workflow_id": "wf_abc123",
  "timestamp": "2024-01-15T10:30:00Z",
  "error": "Action timed out after 30 seconds"
}
```

Your endpoint must respond with a `2xx` status within 10 seconds. If it does not, Regentra retries the delivery up to three times with exponential backoff and marks the notification as failed after all retries are exhausted.

You can verify that a webhook came from Regentra by checking the `X-Regentra-Signature` header against your webhook secret. Find your secret in **Settings → Notifications → [webhook name]**.

<Tip>
  To post Slack alerts without building a custom webhook, use the built-in Slack integration instead. See the [integrations guide](/guides/integrations) for step-by-step setup instructions.
</Tip>
