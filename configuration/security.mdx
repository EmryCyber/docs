Regentra provides several security features you can configure to protect your account, data, and API access. This page covers the controls available to you as a customer.

## API key security

API keys are the primary way your applications authenticate with Regentra. Treat them like passwords.

<Warning>
  Never share API keys or commit them to source code repositories. If a key is exposed, revoke it immediately.
</Warning>

**Rotate keys regularly** — Go to **Settings → API Keys** and generate a new key periodically. Update your application, then revoke the old key once the new one is in use.

**Use scoped keys** — When creating a key, grant only the permissions the application needs. Use read-only keys for monitoring or reporting integrations that don't need to write data.

**Revoke compromised keys** — If you suspect a key has been exposed, go to **Settings → API Keys**, locate the key, and click **Revoke**. Revocation takes effect immediately.

## Two-factor authentication (2FA)

Two-factor authentication adds a second layer of protection to your login. Regentra supports authenticator apps (TOTP) such as Google Authenticator, Authy, or 1Password.

<Steps>
  <Step title="Open security settings">
    Go to **Settings → Security** and click **Enable two-factor authentication**.
  </Step>
  <Step title="Scan the QR code">
    Open your authenticator app and scan the QR code shown in the dashboard.
  </Step>
  <Step title="Enter the verification code">
    Type the six-digit code from your authenticator app to confirm setup.
  </Step>
  <Step title="Save your backup codes">
    Download or copy the backup codes displayed after setup. Store them somewhere safe — you'll need them if you lose access to your authenticator app.
  </Step>
</Steps>

Once enabled, Regentra prompts for a verification code at each login.

## Single sign-on (SSO)

Teams on enterprise plans can configure SSO via SAML, allowing members to log in through your organization's identity provider (such as Okta, Azure AD, or Google Workspace).

To set up SSO for your organization, contact [Regentra support](mailto:support@regentra.io) and provide your identity provider's metadata. The support team will configure SSO and send instructions for your IdP setup.

<Note>
  SSO is available on enterprise plans only. Once SSO is configured, you can optionally enforce it so that all team members must log in through your identity provider.
</Note>

## Audit logs

Regentra keeps a full record of activity in your workspace. Go to **Settings → Audit Logs** to view and filter events.

Logged events include:

- **Logins** — successful and failed authentication attempts
- **API key events** — key creation, rotation, and revocation
- **Workflow changes** — creates, updates, and deletions
- **Settings updates** — changes to workspace, team, and security settings

Audit logs are retained for **90 days**. If you need longer retention, export logs regularly using the API.

## IP allowlisting

You can restrict API access to requests originating from specific IP ranges. Any request from an IP outside the allowlist is rejected, even with a valid API key.

To configure allowlisting, go to **Settings → Security → IP Allowlist** and add the CIDR ranges your servers use.

<Note>
  IP allowlisting is available on paid plans. Ensure you add all IP ranges your applications use before enabling the restriction, or you may lock yourself out of API access.
</Note>

<Tip>
  For details on authenticating API requests securely, see the [API authentication guide](/guides/authentication).
</Tip>
