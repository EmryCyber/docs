Resources are the core data objects in Regentra. Use the Resources API to create, retrieve, update, and delete them programmatically. Every workflow, integration, and automation in Regentra operates on or produces resources.

## Resource object

A resource has the following shape:

```json
{
  "id": "res_abc123",
  "name": "My Resource",
  "type": "task",
  "status": "active",
  "metadata": {},
  "created_at": "2024-01-15T10:00:00Z",
  "updated_at": "2024-01-15T10:30:00Z"
}
```

### Fields

<ResponseField name="id" type="string">
  Unique identifier for the resource. Assigned by Regentra on creation.
</ResponseField>

<ResponseField name="name" type="string" required>
  Human-readable name for the resource.
</ResponseField>

<ResponseField name="type" type="string">
  Category of the resource (for example, `task`, `document`, `asset`). Set at creation and cannot be changed.
</ResponseField>

<ResponseField name="status" type="string">
  Current state of the resource. One of `active` or `archived`.
</ResponseField>

<ResponseField name="metadata" type="object">
  Arbitrary key-value pairs you can attach to a resource for your own use.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the resource was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent update.
</ResponseField>

## Endpoints

### GET /resources

Lists all resources in your account. Results are paginated. Use query parameters to filter by status or type.

#### Query parameters

<ParamField query="page" type="number" default="1">
  Page number to return.
</ParamField>

<ParamField query="per_page" type="number" default="20">
  Number of results per page. Maximum is `100`.
</ParamField>

<ParamField query="status" type="string">
  Filter by status. Accepted values: `active`, `archived`.
</ParamField>

<ParamField query="type" type="string">
  Filter by resource type (for example, `task`).
</ParamField>

#### Example request

```bash
curl "https://api.regentra.io/v1/resources?status=active&page=1&per_page=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Example response

```json
{
  "data": [
    {
      "id": "res_abc123",
      "name": "My Resource",
      "type": "task",
      "status": "active",
      "metadata": {},
      "created_at": "2024-01-15T10:00:00Z",
      "updated_at": "2024-01-15T10:30:00Z"
    }
  ],
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-15T10:30:00Z",
    "pagination": {
      "total": 1,
      "page": 1,
      "per_page": 20,
      "total_pages": 1
    }
  }
}
```

### POST /resources

Creates a new resource.

#### Request body

<ParamField body="name" type="string" required>
  Name of the resource.
</ParamField>

<ParamField body="type" type="string">
  Type of the resource (for example, `task`, `document`). Cannot be changed after creation.
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary key-value pairs to attach to the resource.
</ParamField>

#### Example request

```bash
curl -X POST https://api.regentra.io/v1/resources \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Resource",
    "type": "task",
    "metadata": { "priority": "high" }
  }'
```

#### Example response

```json
{
  "data": {
    "id": "res_abc123",
    "name": "My Resource",
    "type": "task",
    "status": "active",
    "metadata": { "priority": "high" },
    "created_at": "2024-01-15T10:00:00Z",
    "updated_at": "2024-01-15T10:00:00Z"
  },
  "meta": {
    "request_id": "req_def456",
    "timestamp": "2024-01-15T10:00:00Z"
  }
}
```

### GET /resources/{id}

Retrieves a single resource by its ID.

#### Path parameters

<ParamField path="id" type="string" required>
  The ID of the resource to retrieve (for example, `res_abc123`).
</ParamField>

#### Example request

```bash
curl https://api.regentra.io/v1/resources/res_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Example response

```json
{
  "data": {
    "id": "res_abc123",
    "name": "My Resource",
    "type": "task",
    "status": "active",
    "metadata": {},
    "created_at": "2024-01-15T10:00:00Z",
    "updated_at": "2024-01-15T10:30:00Z"
  },
  "meta": {
    "request_id": "req_ghi789",
    "timestamp": "2024-01-15T10:35:00Z"

}
```

### PATCH /resources/{id}

Updates one or more fields on an existing resource. Only the fields you include in the request body are changed.

#### Path parameters

<ParamField path="id" type="string" required>
  The ID of the resource to update.
</ParamField>

#### Request body

<ParamField body="name" type="string">
  New name for the resource.
</ParamField>

<ParamField body="status" type="string">
  New status. Accepted values: `active`, `archived`.
</ParamField>

<ParamField body="metadata" type="object">
  Updated metadata. This replaces the existing `metadata` object entirely.
</ParamField>

#### Example request

```bash
curl -X PATCH https://api.regentra.io/v1/resources/res_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "archived"
  }'
```

#### Example response

```json
{
  "data": {
    "id": "res_abc123",
    "name": "My Resource",
    "type": "task",
    "status": "archived",
    "metadata": {},
    "created_at": "2024-01-15T10:00:00Z",
    "updated_at": "2024-01-15T11:00:00Z"
  },
  "meta": {
    "request_id": "req_jkl012",
    "timestamp": "2024-01-15T11:00:00Z"
  }
}
```

### DELETE /resources/{id}

Permanently deletes a resource. This action cannot be undone.

#### Path parameters

<ParamField path="id" type="string" required>
  The ID of the resource to delete.
</ParamField>

#### Example request

```bash
curl -X DELETE https://api.regentra.io/v1/resources/res_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Response

Returns `204 No Content` on success with no response body.

<Warning>
  Deleting a resource is permanent. Any workflows or integrations that reference the deleted resource may be affected.
</Warning>
