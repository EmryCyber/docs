The Regentra REST API gives you programmatic access to all platform features — manage resources, trigger workflows, configure webhooks, and more. All requests use HTTPS and return JSON. You can use any HTTP client or language to interact with the API.

## Base URL

All API requests are made to the following base URL:

```
https://api.regentra.io/v1
```

Every endpoint path is appended to this base URL. For example, to list resources you send a request to `https://api.regentra.io/v1/resources`.

## API versioning

The current API version is `v1`, included directly in the base URL. Regentra uses URL-based versioning so your integration continues to work even when new versions are released.

Breaking changes — such as removed fields or changed behavior — are introduced only in new major versions (for example, `v2`). Non-breaking additions such as new optional fields or new endpoints may be added to the current version at any time. Subscribe to the [Regentra changelog](https://regentra.io/changelog) to stay informed.

## Request format

For `POST`, `PATCH`, and `PUT` requests, send a JSON body and include the `Content-Type: application/json` header. `GET` and `DELETE` requests do not require a request body.

```bash
curl -X POST https://api.regentra.io/v1/resources \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "My resource", "type": "task"}'
```

<Note>
  All requests must also include your API key in the `Authorization` header. See [Authentication](/api-reference/authentication) for details.
</Note>

## Response format

All API responses are JSON. A successful response wraps the returned data in a `data` field alongside a `meta` object:

```json
{
  "data": { "..." },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-15T10:30:00Z"
  }
}
```

When a request fails, the response contains an `error` object instead:

```json
{
  "error": {
    "code": "resource_not_found",
    "message": "The requested resource could not be found.",
    "request_id": "req_abc123"
  }
}
```

Include the `request_id` value when contacting support — it helps the team trace your request quickly.

## Rate limits

By default, the API allows **1,000 requests per minute** per API key. Every response includes the following headers so you can track your usage:

| Header | Description |
|---|---|
| `X-RateLimit-Limit` | Maximum requests allowed per minute |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp when the window resets |

When you exceed the limit, the API returns a `429 Too Many Requests` response. Back off and retry after the `X-RateLimit-Reset` time. See [Errors](/api-reference/errors) for more on handling this response.

## Pagination

List endpoints return paginated results. Use the `page` and `per_page` query parameters to control which page is returned and how many items it contains:

```bash
curl "https://api.regentra.io/v1/resources?page=2&per_page=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

The response `meta` object includes a `pagination` field with the total count and page details:

```json
{
  "data": ["..."],
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-15T10:30:00Z",
    "pagination": {
      "total": 120,
      "page": 2,
      "per_page": 50,
      "total_pages": 3
    }
  }
}
```

The default value of `per_page` is `20`. The maximum is `100`.
