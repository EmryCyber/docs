## How Regentra works

Regentra is built around a simple, repeatable loop: connect your data, define what should happen, and let Regentra handle the rest. Here's how it works in practice:

1. **Connect your tools and data sources** — Link Regentra to the services your team already uses, such as Slack, GitHub, or Jira, using built-in integrations or the REST API.
2. **Define workflows that react to events or run on a schedule** — Set up triggers based on incoming events, webhooks, or time-based schedules, then attach the actions you want Regentra to execute.
3. **Regentra executes actions and notifies your team** — When a trigger fires, Regentra runs your workflow, updates resources, and sends notifications to keep everyone informed.

## Who is Regentra for?

Regentra is designed for both developers and non-technical team leads. Developers can interact directly with the REST API to build integrations, manage resources programmatically, and trigger workflows from their own systems. Non-technical team leads can use the dashboard to set up and monitor automations, manage team resources, and review activity — no code required. Whether you prefer to script everything or point-and-click, Regentra works the way your team works.
