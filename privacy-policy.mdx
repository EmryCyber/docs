## 5. Data Security

We implement industry-standard security measures to protect your data:

- **Encryption in transit:** All data transmitted over HTTPS (TLS 1.2+) with HSTS enforcement
- **Encryption at rest:** Database encryption via Neon's managed PostgreSQL encryption. Integration credentials encrypted with AES-256-GCM with unique initialization vectors
- **Authentication:** JWT-based sessions with refresh token rotation, TOTP-based multi-factor authentication, brute-force lockout (5 attempts / 15-minute window)
- **Tenant isolation:** Every database query is scoped to the authenticated organization via middleware-enforced filters. Row-level security (RLS) provides defense-in-depth at the database layer
- **Access control:** Role-based access control (RBAC) with 8 granular roles from Read-Only to Super Admin
- **Audit logging:** All sensitive operations (logins, data changes, credential access, report exports) are logged with user, timestamp, and IP address
- **Security headers:** Content Security Policy (CSP), X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy, and Permissions-Policy
- **Input validation:** All API inputs validated with schema enforcement (Zod). HTML content sanitized with DOMPurify. All database queries use parameterized statements
- **Vulnerability management:** Automated dependency scanning in CI/CD pipeline, regular security audits

## 10. Multi-Tenant and MSP Data

Regentra operates a multi-tenant architecture where MSPs manage data for their clients:

- **MSP organizations** are the data controller for their own business data and act as a data processor for their clients' data
- **Regentra** acts as a data processor (or sub-processor) providing the platform
- Each client organization's data is logically isolated at the application and database layers
- MSP administrators can access client data within the scope of their service agreement
- Client end-users accessing the support portal can only see their own tickets and organization data
